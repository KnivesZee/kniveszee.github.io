KMP算法详解

Posted on 2018-11-15 | Post modified: 2018-12-27 | Visitors:
Words count in article: 2,735 | Reading time ≈ 11

今天主要来写一下关于一个学生时代鬼见愁的算法KMP，那么KMP是用来解决那一类问题的呢？

1. 原始问题

返回$str_1$中是否存在子串$str_2$，若存在，则返回$str_1$中$str_2$的起始位置，若不存在，则返回-1

  • 注意子串和子序列的区别：子串是连续的，而子序列不一定连续

tips: 实际中c++/Java中的实现是KMP的加强版实现，但也不可能比$O(n)$快，指数上会更快。

2. 朴素算法(暴力解法)

从$str_1$中逐个对$str_2$进行配对，若出现不匹配，则从$str_1$下个位置继续匹配，直到$str_2$或者$str_1$结束为止

分析：

此方法的时间复杂度为$O(N*M)$,其中$N$为$str_1$的长度指数，$M$为$str_2$长度指数，并且$N>=M$

那么基础方法为什么不行？

因为从任意位置配$str_2$是独立的，没有加速的信息。而KMP是让前面配过的信息指导后面，这是它加速的原因，具体如何加速，后面会讲到。

3. 基础概念

下面先介绍一些KMP算法相关的概念：

3.1 最长前缀和最长后缀的匹配长度

在一个字符串中，最长前缀和最长后缀的匹配长度。比如

求$d$前面的串$abcabc$的最长前缀和最长后缀的匹配长度，这里有一个限定条件：

  • 前缀不能包含最后一个字符
  • 后缀不能包含第一个字符

此时。前缀取3时候$abc==abc$，因此$d$位置信息这里可以记录一个3,这个3的意思是——d前面字符串的最长前缀和最长后缀的匹配长度。

3.2 计算next数组

next数组中的值即$3.1$中介绍的当前字符前的最长前缀和最长后缀的匹配长度。举个$next$数组的例子

$eg: ababac$

默认值如下所示：

  1. 0位置前面没有字符串，默认-1
  2. 1位置上前面就一个字符串，默认为0

计算出来为：
$[-1, 0, 0, 1, 2, 3]$

3.3 next数组加速流程

假设有两个字符串$str_1$和$str_2$：

$str_1$ —- i ——–X

$str_2$ 0———Y

$str_1$的$i$位置开始匹配，一直到$str_1$和$str_2$的$X$，$Y$位置不匹配。

朴素算法的方式为，从$i+1$的位置开始，再与$str_2$的$0$位置开始匹配；而有了$next$信息以后，我们不那么做，因为从i位置开始，$[i,X)$与$[0,Y)$这两个子串一路都是相等的;

我们对$Y$位置求得了一个信息——即最长前缀和最长后缀的匹配长度，所以在$str_2$中假设求得了这个长度为$x$，则$str_2[0,x-1]$和$str_2[Y-x,Y-1]$长度相等且一样，接下来的流程就变成了$str_1[X-x,X-1]$与$str_2[0, x-1]$已经匹配，相对应的从$str_1[Z=x]$开始继续往下匹配即可，$str_2$的位置不变

$eg:$
$str_1$ —abcabct
$str_2$ abcabca

总结来说，就是每一次配对不成功，$str_2$都往右滑动(推)，而且有一段是我们不需要去配对的，从不确定的位置继续往下寻找

3.4 KMP算法加速的实质

一旦$str_1$和$str_2$没有配对上时候，假设这里的$X$和$Y$位置上的字母没有匹配上，Y位置上计算出的next得到的位置是$Z$，对应$str_1$中的$j$位置（[0,Z)和[j,X)这两段是匹配的）。此时开始用$X$和$Z$进行匹配

$str_1$ i——-j——-X
$str_2$ 0——Z——-Y

这件事情的实质是——我们没有否定从$j$位置开头，有可能能匹配出$str_2$这种可能，虽然现在是$X$和$Z$进行比对的，但实质上比对的开头是$j$位置和$0$位置，只是因为$str_1[j,X)$和$str_2[0,Z)$是相等的，于是我们省掉了配对的过程而已；

同时我们否定了从i位置开始到j位置中间这些位置的可能性，因为中间这些位置一律都配不出$str_2$。

为什么i到j之间匹配不到$str_2$呢？

$pf:$

我们假设从i到j之间某位位置开头能匹配到$str_2$

$str_1$ i—k—-j——-X
$str_2$ 0—————Y

假设$i$和$j$之间存在一个$k$位置，从$k$位置出发往后可以匹配出所有的$str_2$,且$next[Y] = X - j$

由此我们可以得到$str_1[k,X)$应该和$str_2$相同长度的前缀可以配（因为既然都从$k$开始匹配了$str_2$全部，那么至少$[k,X]$长度是小于$str_2$的长度的，即$str_2$前面至少会等于$str_1[k,X)$

但是$str_1[i,X) = str_2[0,Y)$

我们就可以得到，$str_2$中[0，Y)就存在了一个长度大于$X-j$的一个最长前缀等与最长后缀,与$next$数组定义相矛盾

so,i到j之间无法匹配到$str_2$

3.7 next数组求法

  1. 0位置 -1
  2. 1位置 0
  3. i位置如何求

假设[0,i-1]位置上已经正确求得

$eg: ababcababct$

其中$str_{i-1} = c$， $str_i = t$

此时：$i-1$的位置的next值为4，这时看长度为4的前缀的下一个字符，

如果它等于i-1位置上的$b$,则$i$位置的next值等于5。那么为什么不可能有更长的，我们做一个假设对$a$来说如果存在一个前缀为6与后缀为6匹配。$i-1$位置肯定是b，那么长度为6的前缀的最后一个字符也必定为$b$,那么当时对$b$求next值的时候，必定为5而不是4，即$i-1$位置上所求得的最长前缀和最长后缀的值是不对的，证毕。

然后就是第二种情况：如果它不等于a,应该怎么求

$eg: ababfababct$

其中$str_{i-1} = c$， $str_i = t$

此时$next[i-1] = 4$（abab）,并且str[4]!=str[i-1]，即$f != c$,此时继续将$abab$划分为前缀$ab$（即求出next[4]=2），再用str[2]的$a$和str[i-1]的$c$进行对比，还是不相等，再往前跳（next[2]=0），$ab$这个串没相应的前后缀了。即用str[0]的$a$和str[i-1]的$c$对比，还是不想等。最后next[0]=-1，此时无法再向前跳了。则结束跳，另next[i]=0即可。

上面的是一直跳到不能再跳的情况，那么如果跳的过程中出现了匹配又如何？

$eg: ababcababak$

这种情况前缀为$abab$, 此时 $c!=a$，$c$前面的字符串为$abab$,匹配的最大前缀为$ab$,此时$a = a$（i位置上的，出现了匹配），则最大前缀长度就等于$c$前面的最大匹配2（next[4]=2）再加上1为3，除此之外不存在更长的了

4. KMP的应用

4.1 question1

假设原始串为$abcabc$，请你生成一个大字符串，只能在原串后面添加，要求此大串的长度是最短的，此大串包含2个原始串，而且开头的位置还要不一样

$eg: abcabc$

结果为：abcabcabc，添加了一个abc

$eg: abcabc$

思路：求得原串得next数组，求完以后，再求一个终止位得next值（△位置处），就求得了整个串的最大前缀和最大后缀，将这两个串叠在一起就是原串，后面再添加上就是答案了

举个例子：

$aaaaa(x)$
求得aaaaa的next为[-1,0,1,2,3]，再多求一个位置x，变为[-1,0,1,2,3,4],我们就知道，后面和前面是可以复用的，叠在一起，最后只用添加一个a即可

4.2 question2 魔性的题目

给你两颗树$T1$,$T2$

问：$T1$中的某个子树是否为$T2$，包含则返回true,否则返回false

求解：将一棵树序列化，变为string的形式，这个string可以唯一的代表这个树，节点为NULL的时候变为#，每个节点之间的值后加上‘_’区分即可

二叉树的序列化，前中后序列都可以

将$T1$序列化为$S1$,$T2$序列化为$S2$，若$S2$是$S1$的子串，则$T2$必定是$T1$的子树

4.3 question 3

问：如何判断一个字符串是否是由子串范式得到的，即一个大字符串是否由一个小字符串重复多次得到的

$eg: 123123123123$ –>true

思路： 实质上就是一个KMP问题，终止条件下前缀和后缀永远是一个整数倍的关系

5. 总结

KMP是70年代的算法，当下流行的大部分都是概率的，贝叶斯，数据挖掘等基于概率的算法，这些基于概率的算法都是不追求最优解的，因为最优解不可达，相当复杂，只能追求优良解。

但基础算法可以提高我们的coding能力，基础算法也可以说是确定算法，有唯一正确解。虽然不是学术，工程的主流，但是它太基础了，一定要搞明白，因为就算是算法工程师也需要考coding。
