1. 什么是Morris遍历

正常来讲，无论是二叉树的递归或非递归遍历方法，都绕不开额外空间复杂度$O(h)$，$h$是二叉树的高度，为什么需要这个$O(h)$呢？因为在一个节点的遍历过程中，你总是要回到这个节点，然后再遍历右子树，而我们二叉树经典的结构没有回到父节点的指针，虽然在有些工程中，会人为的加一根指向父节点的指针，比如红黑树。但是经典的二叉树是没有指向父节点的指针的，这就意味者我门往下走容易，往上走却很困难，此时我们就需要一个栈帮我们回去，递归是递归函数帮你压的栈，非递归是我们自己压的这个栈，但都省不掉这个$O(h)$的额外空间。这也就是我们经典二叉树不管是递归还是非递归都绕不开这个O(h)的额外空间复杂度的原因。

那么Morris遍历是什么呢？——时间复杂度为$O(n)$,但是额外空间复杂度$O(1)$，也可以从下级返回上级完成遍历。它是怎么做的呢？实际上Morris遍历使用的是二叉树上大量的空闲的空间。

什么叫空闲空间？即叶子节点上的空指针，虽然它们都是指向NULL，但是实际上这些left，right指针是占用着空间的。Morris遍历就是利用底层这些空闲的空间完成返回上级的操作的——修改原来的二叉树的结构，让它能够回去，这种统一思路在学术上成为线索二叉树。这里我们不展开线索二叉树，就只说Morris遍历。

2. Morris遍历的流程

下面，我们先来看看Morris遍历的流程：

  • 来到的当前结点叫current，如果current没有左子树，那么current向右移动（current = current->right）
  • 如果current有左子树，就要分情况了，先找到它左子树上最右的节点，记作mostright
    • 如果mostright的右指针是空，让其指向current，然后current向左移动（current = current -> left）
    • 如果mostright的右指针指向current，让其指向空，current向右移动（current = current -> right）

在Morris遍历开始之前，我们先忘记先序，中序，后序这些遍历顺序，心中做到只有Morris序。下面，我们开始举一个例子：

假如有一个高度为3的满二叉树，从第一级到第三级的节点依次为：1，2，3，4，5，6，7

  • 一开始，current指向root（1），我们的标准就从头结点开始。根据我们的标准，current有左子树，先找到它左子树上最右的节点是5，这个5的右指针，指向的是空。则让5->right = cur，然后curent向左移动，指向2
  • 此时current指向2，当前节点有左子树，并且它左子树上最右的节点是4，4的右指针是空，则让4->right = cur(2)，然后current向左移动，来到4
  • 当前current指向4，没有左子树，current向右移动，回到2
  • 然后当前结点左子树是4，最右节点也是4，mostright.right 指向的是current，根据标准，让它指向空，然后current向右移动，来到5
  • 然后由于current没有左孩子，所以向右移动，回到1
  • 然后1的左子树最右节点是5，5.right指向的是current，所以5.right指向空，current向右移动来到3
  • 3有左子树，左子树最右节点为6，其右指针是空，让6->right = 3，然后current来到6
  • 6没左子树，回到3，3左子树最右节点为6，其右指针指向current，所以让其右指针重新指向空，current向右移动来到7
  • current没左子树，current向右移动指向null，整个遍历结束。

现在我们看看什么是Mirros遍历，在上面那个树中，如果一个节点有左子树，Morris遍历能够回到它两次，如果一个节点没有左子树，只到达一个节点一次，而且在我第二次遍历到这个节点的时候，这个节点的左子树都已经遍历完了，它就是这么一种遍历。

所以平时我们的递归版遍历是什么？递归版的遍历就是无论怎么样，只要这个节点不为空，我都可以3次的来到这个节点的过程。那么Morris遍历是什么呢？高度向这个过程致敬，如果我有左子树，我可以来到我自己当前节点两次，如果我没有左子树，我只能来到当前节点一次，但是在遍历我右子树的时候，我无法再第三次回到自己的过程就是Morris遍历。所以它深度的模拟了递归版的过程，只是它是无法向递归那样做到第三次回到自己。

那他到底是怎么标记是第一次还是第二次来到某个节点的呢？它就利用左子树上最右节点的指针指向谁的这件事情标记到底是第一次还是第二次来到这个节点。如果我左子树最右节点是指向空的，我就知道这是我第一次来到这个节点，如果我左子树最右节点是指向我的，我就知道我是第二次回到这个节点的。你递归函数是通过记录每个函数跑过的行号这件事情来区分整个过程的，你是利用函数内部的信息来区分是第一次还是第二次来到这个节点的，而Morris是通过左子树最右节点的右指针来判断这件事情的。

那么，不是完全二叉树也可以吗？可以的。

那么Morris遍历的时间复杂度为什么是$O(n)$的，你会很奇怪，一个节点一直要溜它左子树的右边界，为什么它还是$O(n)$的，证明起来并不困难。

那么在介绍完Morris遍历后，你能不能通过Morris遍历来改先序？有没有想过，为什么这就是我让你忘掉先序，中序，后序遍历的原因。传统的递归遍历实际上是，只要你不为空，都能够来到你3次的这么一个过程。那么，为什么在上面的断点1处埋一个打印，它就变成先序的了呢？有没有想过这个问题。实际上每个节点都会访问3次，你把打印行为放在第一次来到这个节点的时候它就变成了先序遍历，你把打印行为放在第二次来到这个节点的时候，它就变成了中序遍历，你把打印行为放到第三次来到这个节点的时候，它就变成了后序。所以先序，中序，后序是经典遍历的基础上，选择时机不同而产生的东西。

我们Morris是若当前节点有左子树，则来到这个节点两次，如果没有左子树，则来到这个节点一次，那么，我们把Morris遍历过程中，第一次来到一个节点的时候就打印它，是不是就是先序了？没错吧，

5. Morris后序遍历

那么先序，中序都实现了，那如何实现后序？递归中是在第三次返回的时候进行打印，而Morris最多只第二次返回，那它怎么做到后序遍历呢？

还是用上面的那个完全二叉树作为例子。我们把打印时机放在第二次来到一个节点的时候，注意，这个第二次是一个节点首先它必须要两次回到自己，首先它回到叶子结点4，5，6，7就都不行，我们刚刚说了如果一个节点没左子树，它第一次和第二次重在一起了，在我们后序遍历的过程中，我们只关心能够到自己两次的节点。先序和中序就不用担心，因为一个节点没左子树时候，本来就先打印它，再往右边窜。在后序遍历中，我们只关注一个节点能够回到自己两次的节点，然后再逆序的打印它左子树的右边界——什么是右边界，就是当前节点一直顺着右子树往下下直到叶子结点的那一条路。

如果一个节点只能回到自己一次，我们就不要了，这些节点我们是不看的，我们只选它真的有第二次来到自己的时候的这个节点，比如上面那个完全二叉树，Morris的节点遍历顺序是 1，2，4，2，5，1，3，6，3，7。我们把只遍历一次的4，5，6，7去掉，只关心1，2，2，1，3，3。比如我们第二次来到2的时候我们打印什么——逆序打印它左子树的右边界，2左子树的右边界逆序是4，然后第二次来到了1，逆序打印它左子树的右边界的逆序是5，2。然后我门就第二次来到了3，逆序打印它左子树的右边界6，好了，整个打印完以后，在整个函数退出之前，单独打印整棵树的右边界7，3，1。就是后序遍历。

6. 时间复杂度分析

在Morris遍历中，最多只会回到这个节点两次， 假设，当你第一次来到一个节点x的时候，你遍历了一次右边界，第二次来的时候你也遍历了右边界，接下来，你去其他的节点，之前那个右边界是不可能被遍历到的，而且整棵树是可以被右边界分解掉的，所有右边界的节点个数是n个，而每条右边界只会遍历有限几次，所以来到一个节点的时候，遍历一个树的整体代价也就是$O(n)$。

7. 写在最后

二叉树的遍历是基础中的基础，不过你不要把思路限制在先序，中序，后序到底是个啥序列上，那些东西是没有意义的，它就是个处理时机而已，它整体的顺序是和整个代码的程序逻辑是有关的，你可能之前并没有注意到这些东西，你可能没有注意到它是回到自己节点3次的一个行为，在递归过程中就没有注意，所以理解更加宽的遍历顺序中，你就不会写，原因就是你脑子里就只有这些浅显的几个序列，那些玩意儿都是没有用的。
